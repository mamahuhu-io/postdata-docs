## General

<AccordionGroup>
  <Accordion title="What is Postdata?">
    Postdata is a free and open-source API development environment that helps you to quickly test and debug APIs. It is a web-based alternative to Postman, Insomnia, and Paw.
  </Accordion>

## How-to

<AccordionGroup>
  <Accordion title="How to use Postdata?">
    You can use Postdata by visiting [postdata.cn](https://postdata.cn) and start making API requests. You can learn more about Postdata by reading our [Getting started](/guides/getting-started/introduction) guide.
  </Accordion>
  <Accordion title="How to install Postdata?">
    You can install Postdata on your desktop by visiting [postdata.cn](https://postdata.cn) and clicking on the "**Install +**" button. You can also install Postdata on your mobile device by visiting [postdata.cn](https://postdata.cn) and clicking on the "**Install +**" button.
  </Accordion>
  <Accordion title="How to use Postdata on mobile?">
    We're deprecating mobile support for Postdata. You can still use Postdata on mobile by visiting [postdata.cn](https://postdata.cn) and clicking on the "**Install +**" button.
  </Accordion>
  <Accordion title="How to use Postdata on a desktop?">
    Download the desktop app from [Downloads page](https://postdata.cn/downloads) and install it on your desktop. You can also use Postdata on your desktop by visiting [postdata.cn](https://postdata.cn) and clicking on the "**Install +**" button to install the [Progressive Web App](/documentation/clients/web).
  </Accordion>
</AccordionGroup>

## Features

<AccordionGroup>
  <Accordion title="What are the features of Postdata?">
    Postdata has a lot of features that you can use to make your API development experience better. You can learn more about Postdata features by reading our [features](/documentation/features) page.
  </Accordion>
  <Accordion title="How does Collections work?">
    Collections are a way to organize your API requests. You can create a collection by clicking on the "Collections" button on the left sidebar. You can learn more about Collections by reading our [Collections guide](/documentation/features/collections).
  </Accordion>
  <Accordion title="How does Environments work?">
    Environments are a way to store variables that you can use in your API requests. You can create an environment by clicking on the "Environments" button on the left sidebar. You can learn more about Environments by reading our [Environments guide](/documentation/features/environments).
  </Accordion>
</AccordionGroup>

## Bugs

<AccordionGroup>
  <Accordion title="How to report a bug?">
    You can report a bug by clicking on the "Report a bug" button on the left sidebar. You can also report a bug by visiting our [GitHub repository](https://github.com/mamahuhu-io/postdata/issues/new/choose).
  </Accordion>
  <Accordion title="How to report a security vulnerability?">
    You can report a security vulnerability by visiting our [GitHub repository](https://github.com/mamahuhu-io/postdata/security/policy).
  </Accordion>
  <Accordion title="Localhost is not working">
    If you're using Postdata on your local machine, you might need to enable `CORS`. You can learn more about `CORS` by reading our [Set up guide](/documentation/getting-started/setup#locally-served-apis).
  </Accordion>
</AccordionGroup>
